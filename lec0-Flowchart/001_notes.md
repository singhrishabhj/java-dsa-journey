# Flowcharts (Lecture 0)

## What is a Flowchart?

A flowchart is a diagram that represents a step-by-step solution to a problem using symbols and arrows.

---

## Why use Flowcharts?

* Helps in understanding logic before coding
* Makes problem-solving structured
* Reduces errors while writing code

---

## Common Symbols

* **Oval** → Start / End
* **Rectangle** → Process (calculation / instruction)
* **Diamond** → Decision (Yes/No condition)
* **Parallelogram** → Input / Output
* **Arrow** → Direction of flow

Flowchart = Thinking step-by-step before writing code

---

## How to Approach a Flowchart

1. Identify the problem
2. Break it into steps
3. Add decisions where needed
4. Connect everything using arrows

---

## Example Problems

* Making tea ☕
* Finding maximum of 3 numbers

---

## Mistakes I made

* Forgot to label Yes/No in decision
* Skipped arrows between steps
* Used wrong symbols
* Made the flow confusing instead of simple

---

## Key Takeaway

If you can clearly draw a flowchart, coding the solution becomes much easier.
